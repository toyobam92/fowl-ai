Bounce rate — what to fix, by page

GA4 bounce = no 10s+ engagement, no conversion event, no 2nd pageview. 7-day, 154-user sample — bigger than the baseline read, still small.

Homepage
64%
Down from 100% at baseline (small sample both times). Main entry point from Google organic. Add an above-the-fold CTA (subscribe / read latest issue) and internal links to the Jobs Board and Platforms Directory so there's an obvious next click.
Contact
0%
Still the one page that never bounces — submitting the form fires an engagement/conversion event. Replicate that pattern elsewhere: track "apply" clicks, platform-link clicks, and subscribe clicks as events.
All Issues
76%
Down from 100% at baseline. Surface a "start here" or most-popular issue instead of a flat list; a recommended-next-issue link at the bottom of each post keeps sessions going past page one.
AI Jobs Board
53%
Down from 100% at baseline. Add filters, search, or save/apply buttons — interacting with them counts as an engagement event in GA4, lowering bounce independent of time on page.
AI Platforms Directory
74%
Down from 100% at baseline. Same fix as the Jobs Board: add filters/search, or track outbound clicks on platform links as engagement events.
About TYB
80%
Down slightly from 100% at baseline (5 views this week vs. 1). Add a link back to the newsletter or Jobs Board so it isn't a dead end.

Sitewide avg. engagement time dropped to 9s this week (from 17s at baseline) even as bounce rates fell across every page except Contact — a pattern consistent with missing engagement-event tracking outside Contact, not necessarily a real behavior change. TICKET-6 (sitewide event instrumentation) is what would make these numbers trustworthy.
